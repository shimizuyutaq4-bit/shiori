# shimi — ポートフォリオ

## 軸：知識を、"使える形"に整理する。

学ぶ・創るための「体験」を設計する個人開発者です。音楽・言葉・文学・絵と扱う題材はさまざまですが、共通して大切にしているのは、知識を"眺める"で終わらせず、手を動かして"使える"ところまでひとつの体験に落とし込むこと。「見せる」より「整理して、使えるようにする」——それが制作を貫く軸です。

### 自己PR（短縮版）

知識を「見せる」のではなく「使える形に整理する」ことをテーマに、Webアプリやスマホアプリを個人開発しています。音楽理論・語彙・文学など題材は異なりますが、いずれも"学び・創作の詰まりどころ"を見つけ、それを解く体験に組み直すことを大切にしています。代表作は青空文庫×AIの読書アプリ「栞」。単一HTMLで全機能を実装し、収録131作品すべてに手書きのあらすじと実測の難易度を用意しました。実データでの検証と、AIの特性を踏まえた設計（コストと品質の両立）を意識しています。

## 代表作：栞 — AI文豪コンシェルジュ

青空文庫の名作を「**読む → 理解する → 考える → 記録する → 次に出会う**」まで一つで完結させる読書アプリ。単一HTMLファイル製・ビルド不要・PWA対応。収録131作品すべてに手書きのあらすじと実測の難易度をつけています。

公開URL：https://aozora-shiori.netlify.app

**解いた課題**
- 青空文庫のリーダーは多いが「読む」だけで「理解」を助けない → 難語はその場で辞書やAIに聞け、本文を選択して質問できる。
- 名作が多すぎて、どれを読めばいいか分からない → 難易度★と読了目安、「はじめて向き」表示で"今日の気分"で選べる。

**主な機能**
縦書き・ルビ・外字対応／文字を選択して辞書・AI質問・ハイライト・メモ／難易度＆読了目安／読書ログ（しおり・レビュー・気になる）／次の一冊レコメンド／総ルビ（全漢字ふりがな）／読書統計・名文コレクション／PWAでオフライン読書。

**設計で語れること（工夫）**
- **単一HTMLで全機能**：フレームワークを使わず、依存ゼロ・ビルド不要で完結。なぜその構成を選んだかを説明できる。
- **手書きあらすじをAIの文脈に渡す**：作品全文ではなく自作の要約を渡すことで、トークン削減・低コスト・回答品質の向上を同時に実現。AIの特性を理解した設計。
- **難易度は"実測"**：全131作品の本文を取得し、旧仮名/旧字・漢字率・一文の長さから算出。勘に頼らない。
- **総ルビはWeb Workerで**：形態素解析（kuromoji.js）を別スレッドで実行し、スマホでもUIが固まらないようにした。
- **データは全て端末内**：しおり・ハイライト・APIキーも外部に送らないプライバシー設計。

使用技術：HTML / CSS / JavaScript（単一ファイル）、OpenAI API、kuromoji.js、Web Worker、PWA。

## つくり方・大切にしていること

- **体験から逆算する**：機能の羅列ではなく「ユーザーが何をしたいか」から設計する（栞なら"30分で読める一冊を選ぶ"まで含めて考える）。
- **最小構成で作り切る**：単一HTMLなど身軽な構成で、着想からリリースまで一人で完走する。
- **実データで検証する**：難易度は131作品を実測、URLやAPIも一つずつ確認してから組み込む。
- **AIを"設計"として使う**：ただ繋ぐのではなく、要約を文脈に渡す等でコストと品質を両立させる使い方を考える。

スキル：HTML / CSS / JavaScript、Flutter / Dart、Riverpod、AI API活用（OpenAI・Gemini）、PWA / Web Worker、情報設計・UX、和のビジュアルデザイン。
